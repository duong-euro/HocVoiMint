# Definition of Done

A feature is only considered done when all of the following are true:

## Spec Level
- [ ] Feature scope is documented (overview + feature spec)
- [ ] Goals and non-goals are clear
- [ ] Technical spec is concrete (schema, routes, interfaces)
- [ ] Task list is actionable with done criteria
- [ ] Test plan has concrete test cases
- [ ] Codex review completed with findings resolved

## Implementation Level
- [ ] Implementation matches approved spec, or deviations are documented in decision log
- [ ] All tasks in the milestone are checked off
- [ ] All unit tests pass
- [ ] All integration tests pass
- [ ] Manual QA checklist passed
- [ ] No regressions in existing functionality

## Release Level
- [ ] Setup instructions verified
- [ ] Post-release checks passed
- [ ] Known limitations documented
